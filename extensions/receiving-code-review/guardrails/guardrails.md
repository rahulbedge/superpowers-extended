# Receiving Code Review Guardrails

## Think Before Coding

- Read the complete feedback before reacting. Understand the requirement in your own words before implementing.
- Verify every suggestion against codebase reality before accepting it.
- If feedback is unclear, ask for clarification. Do not guess what the reviewer meant.

## Verify External Claims

- Reviewer feedback is a claim, not a fact. Verify against the actual codebase.
- "The reviewer says X, but grepping the codebase shows Y" — report the discrepancy.
- Do not implement a suggestion you haven't verified. Reviewers can be wrong.

## Forbidden Responses

- Never: "You're absolutely right!" — this is performative flattery. Engage with substance directly.
- Never: Blindly implement every suggestion without verification.
- Never: Accept feedback you don't understand. Clarify first.
